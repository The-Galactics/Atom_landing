
# Atom: Multi-Layer Action Agent with Generative AI

Atom is an advanced ecosystem designed to simplify interaction with mobile devices by automating complex workflows through natural language. The project addresses the fragmentation and difficulty of multiple Android customization layers (such as OneUI and HyperOS), centralizing control in an intelligent assistant with self-learning capabilities.

## Project Architecture

The system is built on a robust and modern architecture to guarantee inference latencies of less than 4 seconds:

- **Hexagonal Architecture:** Complete decoupling between business logic, infrastructure adapters, and AI agents.
- **Hybrid Microservices:** Orchestration in Java 21 using Virtual Threads (Project Loom) for massive concurrency and a specialized Python 3.13 bridge for natural language processing.
- **Learning Engine:** Utilization of Dynamic Class Loading to compile and execute new skills generated by the developer AI in real-time.

## Tech Stack

### Frontend (Mobile)
- **Native:** Android SDK (Java 21).
- **UI:** XML with Material Design 3 under a minimalist "Pure Black" (#0A0A0C) approach.
- **Animation:** Lottie for Android for fluid interface transitions.

### Artificial Intelligence
- **NVIDIA NIM:** Inference microservices infrastructure for optimized models.
- **Gemma 4 (Orchestrator):** Multimodal audio-to-intent processing.
- **Qwen 3.5 Coder-VL (Developer):** Interface vision analysis and dynamic code generation.

### Persistence and Data
- **SQL:** PostgreSQL/Supabase for profile management and personalization.
- **Vector:** ChromaDB for storing mathematical representations of workflows.

## Installation and Configuration

To deploy the development environment, the following are required:
1. Java Development Kit (JDK) 21.
2. Python 3.13 with installed dependencies.
3. PostgreSQL configured for user persistence.
4. Constant internet connection for NVIDIA NIM API consumption.
