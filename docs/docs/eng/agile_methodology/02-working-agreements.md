# Working Agreements [Atom]:

This document is primarily designed to attest to the working agreements we have in the Atom project to maintain a constant workflow and a work environment based on respect, efficiency, and focus on the **sprint goals**.

## 1. Communication and Availability:

 - **Daily Scrum:** Will be held every day at **9:00 AM**, in a **face-to-face** format or, in case of absence, through the company's own **Discord**.
 - **Official Channel:** Asynchronous communication will be conducted through the following tools: **Discord** and **WhatsApp**.
 - **Notifications:** If someone is absent or late to a ceremony, they must give at least 10 to 15 minutes' notice.

## 2. Use of Tools and Code:

 - **Scrum Board:** Task status (To Do -> In Progress -> Test -> Done) must be updated immediately after the Daily.
 - **Repository:** All code must be uploaded to a branch following the nomenclature, such as: **feature/task-name**, exclusively in **English**.
 - **Commits:** Commit messages must be descriptive (e.g., **feat: add voice control to the bubble**), exclusively in **English**.

## 3. Decision Making and Impediments:

 - **Consensus:** Technical decisions are made by simple majority. In case of a tie, the [Role/Person] has the casting vote.
 - **Blockers:** If a developer is blocked for more than 30 minutes without progress, the impediment must be escalated to the Scrum Master.

## 4. Respect and Culture:

 - **Focus:** During ceremonies, we avoid using devices unrelated to the meeting.
 - **Constructive Criticism:** We focus on problems, not people.
