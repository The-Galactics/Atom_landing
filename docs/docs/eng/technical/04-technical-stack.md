# Technical Stack Documentation - [Atom]:

This document details the software architecture, implemented technologies, and technical justification of the tools that make up the **Atom** ecosystem, ensuring compliance with performance goals (inferences under 4 seconds), accessibility, scalability, and security.

---

## 1. Development Environment and Methodology
* **Operating System:** Linux.
  * *Benefit:* Provides a highly stable and native development environment for executing Bash scripts, optimizing system resource management during concurrent compilation of multiple microservices and databases.
* **Integrated Development Environment (IDE):** IntelliJ IDEA 2026.
  * *Benefit:* Advanced real-time support for Java 21 features, facilitating deep debugging of virtual threads and Spring Boot framework integrations.
* **Management Tools:** Jira, Git, and Docusaurus.
  * *Benefit:* Jira centralizes Backlog management and Scrum boards, Git with strict branch nomenclature (`feature/*`) ensures version control, and Docusaurus maintains minimalist and easily navigable documentation.

---

## 2. Frontend and UI (Native Mobile)
* **Platform and Language:** Android SDK (Minimum Android 10) using Java 21.
  * *Benefit:* Native development eliminates the latency of cross-platform frameworks, providing direct access to operating system accessibility services and allowing granular control of hardware (microphone, touch event injection).
* **UI and Reactivity:** XML with Material Design 3, View Binding, and RxJava 3.
  * *Benefit:* Implementation of an absolute dark mode (Pure Black `#0A0A0C` optimized for OLED screens) that reduces battery consumption and cognitive load. RxJava 3 efficiently handles real-time asynchronous voice streams, connecting with LiveData for ephemeral transcriptions.
* **Typography and Animation:** Lottie for Android, Geist Serif (Titles), and Lora (Body).
  * *Benefit:* Lottie allows animating the interface's "Core/Atom" with total fluidity (60 fps) using ultra-lightweight JSON files without penalizing the processor. The typography combines an elegant technical profile with high legibility for continuous reading.

---

## 3. Backend and Orchestration Logic
* **Framework:** Spring Boot coupled with LangChain4j.
  * *Benefit:* LangChain4j acts as the perfect orchestrator between enterprise Java code and direct communication with Large Language Models (LLMs), managing memory, tools, and conversation state.
* **Concurrency:** Virtual Threads (Project Loom - Java 21).
  * *Benefit:* Extremely lightweight. They allow the application to maintain thousands of parallel waiting processes (e.g., AI API requests) without exhausting RAM, a critical factor for a real-time assistant.
* **Architectural Design:** Hexagonal Architecture and Dynamic Class Loading.
  * *Benefit:* Decoupling ensures that business logic (the core) remains isolated. Dynamic Class Loading is the engine that allows Atom to "learn": the application compiles, loads, and executes new skills (`.class`) generated by the AI without needing to restart the app, persisting the final code locally for total privacy.
* **Processing Bridge:** Python 3.13.
  * *Benefit:* Acts as a specialized bridge for initial processing and classification algorithms, routing clean data to the Java server.

---

## 4. Data Persistence
* **Relational Database (SQL):** PostgreSQL / Supabase.
  * *Benefit:* Secure, transactional, and robust storage for user profiles and personalization parameters (nicknames, personality). Optimized queries ensure responses in less than 100ms, and data is protected with AES-256 encryption.
* **Vector Database:** ChromaDB.
  * *Benefit:* Fundamental for AI. It stores mathematical representations (vectors) of existing workflows and methods. When a user gives a command, ChromaDB retrieves the exact function by semantic similarity without having to rewrite code.

---

## 5. AI Architecture (NVIDIA NIM Free Tier)
The use of the **NVIDIA NIM** (NVIDIA Inference Microservices) infrastructure allows access to models optimized via TensorRT-LLM with no upfront costs, offering compatible endpoints and incredibly low latency.

* **Orchestrator AI and Native STT: Gemma 4**
  * **Role:** The main multimodal brain that processes audio input, classifies intents, and coordinates agents.
  * **STT Implementation:** Leverages Gemma 4's native capability to process audio tokens directly through the NVIDIA NIM endpoint.
    * *How it works:* The audio stream captured by the Android device is sent via streaming to the model. Gemma 4 performs transcription and semantic understanding in a single step (Audio-to-Intent), eliminating latency from third-party services and saving processing time by avoiding intermediate conversion to plain text.
  * **Justification:** This ensures that inference begins even before the user finishes speaking, meeting the sub-4-second response standard.
* **Developer AI: Qwen (Family 3.5 / Coder-VL)**
  * **Role:** Responsible for reverse engineering, analyzing the interface (Vision), planning the sequence of clicks/gestures, and generating automation code in real-time.
  * **Justification:** Qwen possesses exceptional native multimodal architecture. Its intensive training in code and agent flows allows it to understand a DOM or system visual structure and program a dynamic solution with high precision.

## 6. Text-to-Speech (TTS) Integration for Efficient UX
The architecture employs a **hybrid TTS system** for natural auditory feedback:

* **Main Option: Google Cloud TTS (Neural2 / WaveNet)**
  * **Analysis:** Monthly free tier of up to 1 million characters. Offers highly human-like neural voices.
  * **UX Benefit:** High perceived quality for complex or informative responses.
* **Fallback Option (Ultra-Fast): Android Native TTS (`android.speech.tts`)**
  * **Analysis:** Native integration at the SDK level, 100% free and offline.
  * **UX Benefit:** Response latency under 100ms for quick confirmations, prioritizing task execution if the connection is unstable.
